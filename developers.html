robots.txt

Follow the correct standards (ref: www.robotstxt.org).
Inlcude the URL of the sitemap.xml file if exists.
robots.txt file (all in lower case) needs to be placed in the top-level directory of a web server in order to be useful.
Example: www.domain.com/robots.txt

Set the correct Allow and Disallow commands.

More about this topic?       or Continue?

Sitemap.xml

Sitemap.xml should be automatically generated with a defined frequency or at least every time there is a change in the existing pages (new page or deleted pages).
sitemap.xml will contain only pages allowed to be included in the list. Do not include in the sitemap more than a URL that points to the same resource.
sitemap.mlx has to be located in the top-level directory of a web server OR where it has been specified in the robots.txt file.
sitemap.xml should include the following details:
- Article URLs (this is mandatory)
- Images related to the article (this is recommended)

img : SRC

This is the URL where the image is located. The full URL should contain keywords. The structure should be built in order to allow this setting.


Example:
< img src=”www.domain.com/images/keyword/picture-about-keyword1-and-keyword2.jpg” alt=”Alternative text provided by webmaster” title=”Short description of the image provided by webmaster” >

More about this topic?       or Continue?

img : ALT

Alt parameter should describe effectively the image. Not just a generic text.
Text should be provided to Content Editor / Webmaster, or should be provided a back end where Content Editor / Webmaster can populate the ALT parameter of each image.

Site Map

Site Map is more for Search Engines than humans. It is recommended to create a schematic sitemap that displays the real schema of the web site following the real hierarchy of the structure.
