Sitemap.xml

The best is to get an updated sitemap.xml with an automatic tool that replaces the existing file in order to include also the latest pages of the site. The tool can be launched with a regular frequency (scheduled task) or every time there is a new material to be included (manual task).
There could be many sitemap.xml files for one site (of course with different names), if it is needed a separation.
All sitemap.xml files have to be located in the site root.

Robots.txt

This file contains the rules that Search Engines have to follow on the web pages. It is possible to exclude here a set of pages that are not relevant per the SEO perspective avoiding the crawling/indexing of those pages. This file has to be located in the site root. There should be only one robots.txt file for the entire site. Make sure there is indicated the path to the sitemap.xml file (or files if there are more)

Keep linking depth as short as possible

The less clicks to reach a page the best will be for customer experience and crawlers to get your pages.
1 click: The best
3 clicks: quite ok
5 clicks: useless
